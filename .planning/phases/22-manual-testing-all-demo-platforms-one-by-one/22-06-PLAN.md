and serves an `index.html` that loads `web.js`. The dev server is a long-running process —
must be backgrounded with a timeout, or the user kills it with Ctrl+C after observation.

Browser console signal:
- `console.log` calls from kotlin (println / Logger.log) appear in DevTools Console
- A passing run shows non-INVALID values streaming roughly per second
