  Layer 1 — kamper/api/        Shared contracts (Info, Config, Watcher, Performance, Logger)

  androidMain, androidTest, commonMain, commonTest, iosMain, jsMain, jvmMain, jvmTest, macosMain, tvosMain, wasmJsMain
  (8 platform main sets: android, ios, jvm, macos, tvos, js, wasmJs + common)

Gradle command reference (from RESEARCH.md, verified against TESTING.md + .claude/settings.json):
  ./gradlew :kamper:modules:<name>:jvmTest   — Fast JVM-only test, NO device, primary path for Claude
  ./gradlew :kamper:modules:<name>:test      — All unit tests (incl. androidUnitTest via Robolectric/MockK), no device
  ./gradlew :kamper:api:test                 — API contracts
  ./gradlew :kamper:engine:test              — Engine
  ./gradlew test                             — All modules all unit tests
  ./gradlew detekt                           — Static analysis (zero-tolerance, maxIssues: 0)
  ./gradlew connectedAndroidTest             — INSTRUMENTED — REQUIRES emulator/device — DO NOT run in Claude sessions
  ./gradlew assembleXCFramework              — iOS/macOS XCFramework (compiles on host, only when publishing)

Commit convention (from RESEARCH.md, verbatim):
  Format:  <type>(<scope>): <short description>
  Types:   feat | fix | chore | docs | test | refactor
  Scopes:  cpu | fps | memory | network | issues | jank | gc | thermal | engine | api | ui | build | deps | phase<N>
  Examples:
    feat(cpu): add thermal throttle detection
    fix(engine): guard against duplicate module install
    chore(deps): bump coroutines to 1.10.1
    docs(api): update Watcher KDoc
  Rules:
    - No emojis
    - No "resolves #N" footers
    - No develop branch — main is the integration branch
    - Squash merge when merging executor worktree branches
    - Description is imperative mood, lowercase, no trailing period

Module 4-class structure (from CONVENTIONS.md):
  {Name}Info       — data class : Info, has companion INVALID sentinel
  {Name}Config     — data class : Config, has @KamperDslMarker Builder object + DEFAULT companion val
  {Name}Watcher    — internal class : Watcher<{Name}Info>, polling loop with `while (isActive)`
  {Name}Performance— internal class : Performance<{Name}Config, IWatcher<{Name}Info>, {Name}Info>
  expect val {Name}Module: PerformanceModule<{Name}Config, {Name}Info>   in commonMain/Module.kt
  actual val {Name}Module: PerformanceModule<{Name}Config, {Name}Info>   in EACH of 7 platform main source sets

ADRs (from .planning/codebase/adr/):
  ADR-001: Plugin architecture (manual install pattern)
  ADR-002: KMP expect/actual pattern
  ADR-003: Listener pattern
  ADR-004: No breaking changes (API freeze)

Safety rule (D-06 from CONVENTIONS.md):
  Every platform-specific call wrapped in try/catch. Exceptions logged + absorbed.
  Kamper must NEVER crash the host app. INVALID sentinel returned on failure.
