**Android verification (required):**
1. Connect an Android device or start an emulator
2. Install: `adb install demos/flutter/build/app/outputs/apk/debug/app-debug.apk`
   (or `flutter install` from `demos/flutter/` directory)
3. Launch the app. Verify:
   - App title shows "K|Android|FL"
   - 10 tabs visible: CPU, GPU, FPS, MEMORY, EVENTS, NETWORK, ISSUES, JANK, GC, THERMAL
   - Running indicator dot is gray (stopped state)
   - START button has blue background
4. Tap START. Verify:
   - Running indicator turns green
   - STOP button appears with gray background
   - CPU tab: 5 progress bars begin animating with percentage values
   - FPS tab: large number changes and shows fps value
   - MEMORY tab: heap and RAM bars show values
   - NETWORK tab: RX/TX values appear (may be 0 if no traffic)
   - GPU tab: shows utilization % (may be 0 or "—%" if device not supported)
5. Tap ISSUES tab and then trigger an ANR or wait 30s. Or just verify the "No issues detected" empty state is visible.

   - Tap "USER_LOGIN" preset — it should appear in the events list
   - Type a custom name and tap LOG
   - Tap "VIDEO_PLAYBACK" — should show "RECORDING…" state, tap again to end
7. Tap STOP. Verify running indicator turns gray and data stops updating.
8. In CPU tab footer: tap "START CPU LOAD" — verify CPU usage increases.
