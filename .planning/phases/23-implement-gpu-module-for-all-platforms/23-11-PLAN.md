GpuInfo (commonMain): `data class GpuInfo(utilization: Double, usedMemoryMb: Double, totalMemoryMb: Double)`
- INVALID = GpuInfo(-1.0, -1.0, -1.0)
- UNSUPPORTED = GpuInfo(-2.0, -2.0, -2.0)

iosMain/tvosMain `actual val GpuModule` (per 23-04-SUMMARY): repository returns UNSUPPORTED unconditionally.
This means listeners receive UNSUPPORTED on every poll. UI should render the grayed-out state.

Theme palette (existing, both Theme.kt files): MAUVE for valid GPU primary color (D-10 mauve accent),
MUTED for UNSUPPORTED label (D-12 grayed). All 3-channel UIColor/NSColor constructors already provided.
</interfaces>

<apple_pattern_reference>
ThermalViewController.kt (iOS) is canonical for GpuViewController.kt (iOS). Key elements:
- `class GpuViewController : UIViewController(nibName = null, bundle = null)`
- Three lateinit UILabel fields: `bigLabel` (primary metric), `unitLabel` ("GPU usage %"), `memoryLabel` ("Memory:  —")
- viewDidLoad creates labels with monospaced large font, applies translatesAutoresizingMaskIntoConstraints, sets up NSLayoutConstraint vertical stack
- **OMIT** stress button + ActionTarget + stressJobs list (no GPU stress simulator)
- **OMIT** simHintLabel ("Not available on simulator") because GPU on iOS is intentionally UNSUPPORTED everywhere — replace with a permanent hint label "Bridge: IOKit-restricted on iOS" for clarity
- `update(info: GpuInfo)`:
  - if `info == GpuInfo.INVALID` → return
  - if `info == GpuInfo.UNSUPPORTED` → bigLabel.text="Unsupported"; bigLabel.textColor=Theme.MUTED; memoryLabel.text="Memory:  N/A"
  - else (defensive): bigLabel.text="${"%.1f".format(info.utilization)}%"; bigLabel.textColor=Theme.MAUVE; memoryLabel.text formatted per D-02 partial-data rules
  - Defensive utilization < 0 guard — show "—%"

ThermalViewController.kt (tvOS) is canonical for GpuViewController.kt (tvOS). Identical structure to iOS but with:
- Larger fonts: 100pt bigLabel (vs 48pt on iOS)
- Larger padding: 80.0 (vs 20.0 on iOS)
- All labels use Theme.LABEL_FONT (already 13pt+ for tvOS distance)
- Segmented control on tvOS lives in Main.kt; this ViewController only renders content for its tab.

The tvOS file freeCompilerArgs include `-Xpartial-linkage=enable` for KN UIKit klib gaps — already set in tvos/build.gradle.kts; do NOT modify.
