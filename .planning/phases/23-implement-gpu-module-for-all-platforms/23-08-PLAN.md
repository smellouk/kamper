From libs/modules/gpu/src/commonMain/kotlin/com/smellouk/kamper/gpu/GpuInfo.kt:
```kotlin
data class GpuInfo(
    val utilization: Double,    // 0..100, or -1.0 (INVALID), or -2.0 (UNSUPPORTED)
    val usedMemoryMb: Double,   // MB, or -1.0 (unknown), or -2.0 (UNSUPPORTED)
    val totalMemoryMb: Double   // MB, or -1.0 (unknown), or -2.0 (UNSUPPORTED)
) : Info {
    companion object {
        val INVALID     = GpuInfo(-1.0, -1.0, -1.0)
        val UNSUPPORTED = GpuInfo(-2.0, -2.0, -2.0)
    }
}
```

From libs/modules/gpu/src/androidMain/kotlin/com/smellouk/kamper/gpu/Module.kt (per 23-03-SUMMARY):
- `actual val GpuModule` exists; on Adreno devices reads `/sys/class/kgsl/kgsl-3d0/gpu_busy_percentage`,
  on Mali devices falls back to `/sys/class/devfreq/`. Memory fields are -1.0 on Android (no public path).
- Most stock Android devices return `GpuInfo.INVALID` from a kgsl read failure → listener filters this out.
- A small subset of Adreno devices return real utilization with usedMemoryMb=-1.0 (D-02 partial data).
