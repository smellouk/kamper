## Architecture


### Per-Module Config Parsing (D-08)

```kotlin
private fun ReadableMap?.flag(key: String): Boolean {
    if (this == null || !this.hasKey(key)) return true       // absent → default true
    return if (this.getType(key) == ReadableType.Boolean) {  // T-12-11: type guard
        this.getBoolean(key)
    } else {
        true                                                 // non-boolean → default true
    }
}
```

Each of the 8 modules is gated: `if (config.flag("cpu")) install(CpuModule)`

### INVALID Guards Pattern

| Listener | Guard |
|----------|-------|
| CpuInfo | `if (info == CpuInfo.INVALID) return@addInfoListener` |
| FpsInfo | `if (info == FpsInfo.INVALID) return@addInfoListener` |
| MemoryInfo | `if (info == MemoryInfo.INVALID) return@addInfoListener` |
| NetworkInfo | `if (info == NetworkInfo.INVALID \|\| info == NetworkInfo.NOT_SUPPORTED) return@addInfoListener` |
| IssueInfo | No guard (matches existing KamperModule.kt pattern) |
| JankInfo | `if (info == JankInfo.INVALID) return@addInfoListener` |
| GcInfo | `if (info == GcInfo.INVALID) return@addInfoListener` |
| ThermalInfo | `if (info == ThermalInfo.INVALID) return@addInfoListener` |

## Legacy Bridge Anti-patterns Absent

The following legacy artifacts from `KamperModule.kt` are NOT present in `KamperTurboModule.kt`:

- `@ReactMethod` annotations — replaced by abstract method overrides
- `DeviceEventManagerModule.RCTDeviceEventEmitter` import — replaced by Codegen `emitOnXxx()`
- `fun addListener(eventName: String?)` stub — TurboModule EventEmitter<T> doesn't need it
- `fun removeListeners(count: Int)` stub — same reason
