    ```markdown
    # kamper-demo

    Companion Android app for the [Kamper](https://github.com/smellouk/kamper) performance monitoring library — induces every bottleneck Kamper monitors so you can clone, run, and watch detection happen in real time.

    Each screen pairs with an article in the [Kamper performance series](https://medium.com/) (links published as articles ship).

    ## Quick Start

    ```bash
    git clone https://github.com/smellouk/kamper-demo
    cd kamper-demo
    ./gradlew :app:assembleDebug
    ./gradlew :app:installDebug   # requires connected device or emulator
    ```

    Requires JDK 17 and Android SDK (API 21+ device or emulator).

    ## What You'll See

    Launch the app. The Kamper UI debug overlay appears as a draggable chip — tap to expand the panel showing live metrics from all 8 modules (CPU, FPS, Memory, Network, Jank, GC, Thermal, Issues).

    The chip color escalates as bottlenecks trigger: green → yellow → orange → red.

    ## Screens

    > **Thermal screen** requires a physical Android device. Emulators report `THERMAL_STATUS_NONE` because they have no physical thermal sensor.

    ## Architecture

    - **Central-install pattern** — `MainActivity` installs all 8 Kamper modules once. Each tab Fragment exposes a public `update(info)` method that the Activity-level listener calls.
    - **Demo-only crash handling** — the demo uses `chainToPreviousHandler = false`. Production apps must use `true` to preserve any pre-existing crash handler.
    - Kamper UI overlay is auto-installed by the `ui-android` artifact's ContentProvider — no app-side initialization required.

    ## Develop Against a Local Kamper Checkout

    By default this repo consumes Kamper from Maven Central (`com.smellouk.kamper:engine:VERSION` etc., declared in `gradle.properties`).

    To develop against a local Kamper monorepo checkout sitting in a sibling directory:

    1. Clone Kamper alongside this repo (so `../kamper` resolves):
       ```bash
       cd ..
       git clone https://github.com/smellouk/kamper
       cd kamper-demo
       ```
    2. Add to `local.properties` (gitignored):
       ```properties
       useLocalKamper=true
       ```
    3. Re-run Gradle. `settings.gradle.kts` will composite-build `../kamper` and substitute Maven coordinates with project paths.

    ## Develop Against Local Kamper

    To develop a Kamper change in tandem with a kamper-demo change:

    1. Clone Kamper as a sibling: `git clone https://github.com/smellouk/kamper ../kamper`
    2. Create `local.properties` in this repo with `useLocalKamper=true`
    3. Gradle will composite-build `../kamper` and substitute every `com.smellouk.kamper:*` artifact with its project-path equivalent — see `settings.gradle.kts`.

    Reset by removing the `useLocalKamper` line (or deleting `local.properties` entirely) — the build returns to consuming published Maven artifacts.

    ## Adding a New Screen

    1. Create the package under `app/src/main/kotlin/com/smellouk/kamperdemo/<topic>/`
    2. Add a `<Topic>Screen.kt` Fragment with a public `fun update(info: <Topic>Info)` method
    3. Add the layout under `app/src/main/res/layout/screen_<topic>.xml`
    4. Wire it in `MainActivity.kt`: install the module in `setupKamper()` and add an `addInfoListener<<Topic>Info> { ... <topic>Screen.update(it) }` line
    5. Add an entry to the README Screens table
    6. Add a tab title to `MainActivity.tabTitles`

    ## Code Style

    - Kotlin official style (`kotlin.code.style=official` in `gradle.properties`)
    - Mirror the existing screen file structure (lifecycle-scoped view nullability, `activity?.runOnUiThread { ... }` UI dispatch, Catppuccin Mocha colors from `colors.xml`)

    ## Commit Messages

    Conventional Commits (matching the [Kamper repo](https://github.com/smellouk/kamper) convention):

    ```
    <type>(<scope>): <description>
    ```

    Allowed types: `feat`, `fix`, `chore`, `docs`, `test`, `refactor`.
    Allowed scopes: `cpu`, `fps`, `memory`, `network`, `issues`, `jank`, `gc`, `thermal`, `ui`, `build`, `deps`, `ci`.

    Example: `feat(cpu): add per-core CPU bar visualization`

    No emojis. No `resolves #N` footers. Squash merge to `main`.

    ## CI

    `.github/workflows/build.yml` runs `:app:assembleDebug` on every push and pull request to `main`. The build must pass before merging.
